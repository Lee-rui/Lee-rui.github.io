《图形用户界面》

发表于 2016-09-22   |  

​ 图形用户界面GUI（Graphical User Interface）编程主要有一下几个特征

  • 图形界面对象及其框架（图形界面对象之间的包含关系）

  • 图形界面对象的布局（图形界面对象之间的位置关系）

  • 图形界面对象上的事件响应（图形界面对象上的动作）

    Java的图形界面开发中有两种可使用的技术：AWT、Swing。AWT中大量使用了windows的系统函数，不是使用Java开发的，所以是重量级。Swing是由Java来实现的用户界面类，可以在任意的系统平台上工作，属于轻量级，但是在Swing中仍然大量使用了AWT的概念。

1. AWT简介

​ AWT，抽象窗口工具包，所有工具类都保存在java.awt包中，此包中的所有操作类可用来建立与平台无关的图形用户界面（GUI）的类，这些类又被称为组件（Components）

​ 在整个AWT包中提供的所有工具类主要分为以下3种

  • 组件：Component
  • 容器：Container
  • 布局管理器：LayoutManager

1.1 组件

​ 组件就是我们常说的一个个按钮、标签、菜单等

​ 在AWT包中，所有的组件类（如按钮、文本框等）都是从Component和MenuComponent扩展而来的，这些类会继承两个类的公共操作。

1.2 容器

​ 所有的AWT组件都应该放到容器中，并可以设置其位置、大小等，所有的容器都是Component的子类。

1.3 布局管理器

​ 布局管理器可以使容器中的组件按照指定的位置进行摆放，另外一个就是，即使容器改变了大小，布局管理器也可以准确地把组件放到指定地位置。在AWT中所有地布局管理器都是LayoutManger的子类。

2. Swing简介

​ Swing是以AWT平台为基础构建起来的新组件，所有的Swing都保存在javax.swing包中。所有的组件都是从JComponent扩展出来的。此类实际上是java.awt.Component的子类。

​ JComponent类几乎是所有Swing组件的公共超类。

3. 基本容器：JFrame

4. 标签组件：JLabel

4.1 认识JLabel

​ JLabel组件表示的是一个标签，本身是用于显示信息的，一般情况下是不能直接更改其显示内容的。创建完的Label对象可以通过Container类中的add()方法加入容器中

​ JLabel常用的方法和常量：

序号 方法及常量 类型 描述

2 public static final int CENTER 常量 标签文本居中对齐
3 public static final int RIGHT 常量 标签文本右对齐
4 public JLabel() throws HeadlessException 构造 创建一个JLabel对象
5 public JLabel() throws HeadlessException 构造 创建一个标签并指定文本内容，默认为左对齐
6 public Label(String text,int alignment) throws HeadlessException 构造 创建一个标签并指定文本内容以及对齐方式，可以使用JLabel.LEFT、JLabel.RIGHT、JLabel.CENTER 3个值
7 public JLabel(String text,Icon icon,int horizontalAlignment) 构造 创建具有指定文本、图像和水平对齐方式的JLabel对象
8 public JLabel(Icon image,int horizontalAlignment) 构造 创建具有指定图像和水平对齐方式的JLabel实例
9 public void setText(String text) 普通 设置标签的文本
10 public String getText() 普通 取得标签的文本
11 public void setAlignment(int alignment) 普通 设置标签的对齐方式
12 public void setIcon(Icon icon) 普通 设置指定的图像

例：使用一个标签

​ 以上的标签内容只是使用了默认的字体及颜色显示，如果现在要更改使用的字体，则可以直接使用Component类中定义的以下方法

​ public void setFont(Font f)

​ 在设置时使用了java.awt.Font类表示字体。

例：设置标签的显示字体、大小、背景颜色

6. 布局管理器

​ 在Swing中使用的所有布局管理器都是可以实现LayoutManager接口，在Swing中主要使用了以下5中常见的布局管理器：FlowLayout、BorderLayout、GridLayout、CardLayout、绝对定位。

6.1 FlowLayout

​ FlowLayout属于流式布局管理器，使用此种布局方式会使所有的组件像流水一样依次进行排列，FlowLayout类的常用方法及常量如下表：

​ FlowLayout类的常用方法及常量

序号 方法及常量 类型 描述
1 public static final int CENTER 常量 居中对齐
2 public static final int LEADING 常量 与容器的开始端对齐方式一样
3 public static final int LEFT 常量 左对齐
4 public static final int RIGHT 常量 右对齐
5 public static final int TRAILING 常量 与容器的结束端对齐方式一样
6 public FlowLayout() 构造 构造一个新的FlowLayout，居中对齐，默认水平和垂直间距式5个单位
7 public FlowLayout(int align) 构造 构造一个FlowLayout，并指定对齐方式

例：设置FlowLayout

6.2 BorderLayout

​ BorderLayout将一个窗体的版面划分成东、西、南、北、中5个区域，可以直接将需要的组件放到这5个区域中，

​ BorderLayout的常用方法及常量如下：

序号 方法及常量 类型 描述
1 public static final String EAST 常量 将组件设置在东区
2 public static final String WEST 常量 西区
3 public static final String SOUTH 常量 南区
4 public static final String NORTH 常量 北区
5 public static final String CENTER 常量 中区
6 public BorderLayout() 构造 构造没有间距的布局器
7 public BorderLayout(int hgap,int vgap) 构造 构造有水平和垂直的布局器

例：设置BorderLayout

6.3 GridLayout

​ GridLayout布局管理器式以表格的形式进行管理的，在使用此布局管理器时必须设置显示的行数和列数

​ 常用方法：
